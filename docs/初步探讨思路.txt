金融行业最关注的五大核心内容
金融从业者每天处理的信息，我浓缩成五大类：
1️⃣ 公司基本面信息

财务数据：营收、净利润、现金流、毛利率、ROE、资产负债率
公告：业绩预告、重大合同、股东减持、分红方案、诉讼
研报：券商/独立研究机构的深度/点评报告
招股说明书（IPO场景）、债券募集说明书（债券场景）

使用场景： 分析师写一份个股研究报告、基金经理调仓前的尽调
2️⃣ 宏观与行业信息

宏观数据：GDP、CPI、PPI、PMI、社融、M2、利率、汇率
行业数据：出货量、价格指数、开工率、库存周期
政策文件：央行货币政策、财政部发文、行业监管新规
重大事件：地缘冲突、疫情、极端天气对行业的影响

使用场景： 宏观研究员写策略报告、投资经理判断行业轮动
3️⃣ 市场交易信息

行情：股票/基金/债券/期货/汇率/大宗商品实时价格
资金面：北向资金、融资融券、ETF申赎、大宗交易
情绪指标：涨跌家数、成交量、波动率、换手率
异动监控：涨停/跌停板、大单净流入

使用场景： 交易员盘中操作、量化策略研发
4️⃣ 客户与业务信息（银行/保险/券商中后台）

客户画像：KYC信息、资产配置、风险偏好、交易习惯、生命周期
产品信息：理财产品、保险产品、基金池、信贷产品
业绩数据：AUM（管理规模）、净值、收益率、回撤、夏普比率
客户反馈：投诉、咨询、满意度

使用场景： 客户经理做配置方案、产品经理设计新产品
5️⃣ 合规与风险信息

监管规则：银保监/证监会/人行发文、《资管新规》、《销售适当性办法》
内控规则：公司内部审批流程、黑名单、白名单、敏感词
风险信号：征信报告、反洗钱监控、关联方交易、异常行为
舆情监控：负面新闻、社交媒体舆情

使用场景： 合规审查、风控审批、反洗钱筛查
FinPilot 的功能映射（重要）
基于上面的框架，我把你的"三合一"重新设计成四个功能模块，每个都对应一个真实的信息类别和岗位：
🔧 模块一：个股/行业深度分析（对接公司基本面信息）
目标用户： 投研分析师、买方研究员
核心能力：

上传/指定一家公司 → Agent自动抓取年报+研报+公告 → 输出标准化分析摘要
支持自然语言追问："这家公司的现金流为什么连续两年下滑？"
支持同行对比："把宁德时代和比亚迪的毛利率趋势画个图"

你的技术落点： RAG（长文档处理）+ Function Calling（调用财务指标计算）+ ANIMA（多轮追问规划）

🔧 模块二：销售话术/营销物料合规审查（对接合规信息）
目标用户： 客户经理、合规专员、产品经理
核心能力：

粘贴一段产品宣传文案 → Agent基于监管规则库审查 → 标红违规点 + 给出合规修改建议
例如识别："保本保息""年化收益10%起"这类违规表述
输出合规评分 + 修改版本

你的技术落点： Model Court多Agent共识（多个Agent从不同监管维度审查，比如"适当性""风险揭示""收益表述"各一个Agent）+ RAG（监管规则库）
这是你Demo的灵魂功能，因为它把你已经发布的PyPI包直接商用化了，面试时可以说"这不是一个新做的Demo，这是我开源框架在金融场景的落地"。

🔧 模块三：投研晨报生成（对接市场+宏观+行业信息）
目标用户： 投研助理、基金经理助理、新入职分析师
核心能力：

早上8点 → Agent自动汇总昨日收盘后的公告、研报、新闻、海外行情 → 生成结构化晨报
按板块分类（科技/消费/金融/周期）、按重要性排序
支持个性化订阅（"我只关注新能源和半导体"）

你的技术落点： 多文档处理 + 实体抽取 + 自动分类 + 摘要生成

🔧 模块四：客户画像与配置建议（对接客户业务信息）
目标用户： 银行理财经理、券商投顾、保险代理人
核心能力：

录入客户基本信息 + 对话记录 → Agent提取KYC要素 → 生成客户画像
基于画像给出**"配置逻辑"**（不是具体产品推荐，避开合规红线）
例如：风险偏好中等 + 子女教育金目标 → 建议"债基+指数基金"的配置逻辑

你的技术落点： 结构化信息抽取 + 规则引擎 + Agent决策
⚠️ 重要合规提示： 这个模块只给"配置框架建议"，不推荐具体产品，避免踩"非持牌投顾"的红线。
