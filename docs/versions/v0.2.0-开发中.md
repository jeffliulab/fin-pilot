## 1. 一句话目标

把 v0.1 的"个股深度分析"升维到"**行业横向对比**"—— 用户选一组公司 × 一组问题，AI 并发跑出 **Generative Grid**（5 公司 × 8 问题 = 40 个 cell），每个 cell 是流式生成的精炼答案 + 引用。形态对齐 Hebbia Matrix（[research/competitor-landscape.md §Q1 Hebbia](../research/competitor-landscape.md)）。

## 4. 架构

### 4.1 Grid orchestrator（关键创新）

按 [agent-rules / principles/architecture.md](https://github.com/jeffliulab/agent-rules) "注册表 / 工厂" 模式：

```
POST /api/v1/industries/grid/stream
  body: {industry_id, ticker_list[5], question_list[8]}
  ↓
grid_orchestrator.run(industry_id, tickers, questions):
  # 5 × 8 = 40 个 cell concurrent
  cells = []
  for ticker in tickers:
    for question in questions:
      cell_id = f"{ticker}-{q.idx}"
      cells.append(asyncio.create_task(
        single_cell(ticker, question, cell_id)
      ))
  
  # 流式 yield 每个 cell 的进度
  async for cell_done in asyncio.as_completed(cells):
    yield cell_done  # → SSE Vercel AI SDK Data Stream Protocol
```

**协议扩展**（在现有 0:/2:/d:/3: 之上加新 type）：
```
2:[{"type":"cell_start", "cell_id":"600519-0"}]
2:[{"type":"cell_delta", "cell_id":"600519-0", "delta":"茅"}]
2:[{"type":"cell_delta", "cell_id":"600519-0", "delta":"台 Q1..."}]
2:[{"type":"cell_done", "cell_id":"600519-0", "citations":[...]}]
2:[{"type":"cell_error", "cell_id":"600519-0", "msg":"..."}]
d:{"finishReason":"stop", ...}
```

前端 `useGridStream` hook 解析后按 `cell_id` 分发到对应 cell 组件 → 流式更新。

### 4.2 限流 / 成本控制

- 40 个并发 LLM 调用，token 烧得快。**默认 LLM_PROVIDER=openai + OPENAI_MODEL=gpt-4o-mini**（每 cell 约 200-500 token，40 cell ≈ 0.01 USD）
- backend 加 `MAX_CONCURRENT_CELLS = 8`（asyncio.Semaphore），保护 LLM API rate limit
- 失败 cell 优雅降级：`cell_error` 而非整个 grid 失败

### 4.3 行业数据源

AKShare 的 `stock_board_industry_*` API：
- `stock_board_industry_name_em()` —— 列出所有行业（板块名 + 板块代码）
- `stock_board_industry_cons_em(symbol="白酒")` —— 该行业的成员公司（含 ticker）

容错：API 偶发改版 / 限流 → 提供静态 fallback 字典覆盖最常见 20 个行业。

## 6. 演示故事

1. 用户进入 `/industry`
2. 选行业"白酒" → 显示 10 家成员公司（贵州茅台 / 五粮液 / 山西汾酒 ...）
3. 用户多选 5 家
4. 选 8 个预设问题（默认全选）
5. 点"生成 Grid"
6. 中工作区出现 **5 行 × 8 列 网格**，每个 cell 显示 spinner
7. 流式：cell 一个一个填充（concurrency=8，约 30-60 秒填满）
8. 每个 cell 角标 `[N]` → 点击抽屉打开原文
9. 点 cell 卡片 → 抽屉展开"详细推理"
10. 右上角"导出 Excel" → 下载 .xlsx

## 10. 风险 + 缓解

| 风险 | 缓解 |
|---|---|
| 5×8 并发 LLM 烧钱 | 默认 gpt-4o-mini + Semaphore(8) 限流 + 用户预估提示 |
| AKShare 行业 API 改版 / 限流 | 静态 fallback 字典覆盖 20 个常用行业 |
| 部分 cell 失败导致 grid 缺角 | cell_error 优雅降级，单 cell 显示 retry 按钮，整 grid 不挂 |
| Grid UI 在窄屏不友好 | 强制最小宽度 1200px + 移动端提示"建议桌面查看" |
| 用户期望"行业指数" / "行业新闻" | v0.2 明确不做（v0.3 市场模块），UI 上写清楚 |
