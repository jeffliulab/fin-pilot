## 2026-04-24（续 — Day 2）

### 完成

**Day 2 — 后端：data providers**
- 写 `backend/interfaces.py` —— 跨层契约的核心：`Citation` / `CompanyCard` / `FinancialMetric` / `FinancialStatements` / `Announcement` / `ResearchReport` / `MarketDataProvider` Protocol / `DataSourceError`
- 写 `backend/constants.py` —— limits（公告 20 / 研报 30 / 季度 4）+ EDGAR endpoints + AKShare 前缀映射
- 写 `backend/repositories/market/akshare_provider.py` —— A 股 provider，三个公开方法对应 `stock_financial_abstract` / `stock_notice_report` / `stock_research_report_em`，列名用容错 lookup 应对中英混杂；研报**只入元数据**符合合规策略
- 写 `backend/repositories/market/edgar_provider.py` —— 美股 provider：ticker→CIK 进程级缓存（`company_tickers.json`），companyfacts 抽 Revenue / NetIncomeLoss / OperatingCashFlow / StockholdersEquity 并算 ROE，submissions 过滤 10-K/10-Q/8-K/20-F/S-1/DEF 14A；统一 150ms sleep 守 SEC 7 req/s 安全裕度
- 写 `backend/repositories/market/factory.py` —— `get_provider(ticker)` 注册表式分派；裸 6 位 A 股 code 容错（market_types.py 默认会归 UNKNOWN）；非 A/US 抛 `UnsupportedMarketError`
- 写 3 个测试文件：`test_market_factory.py`（13）+ `test_akshare_provider.py`（10）+ `test_edgar_provider.py`（12）= 35 tests，全 pass（在 `/tmp/finpilot-test-venv` 里跑通，pytest 8.x + pandas 2.x + httpx 0.27）

### 决策

- **`MarketDataProvider` Protocol 放 `interfaces.py` 而非 `repositories/market/base.py`**：agent-rules 的 "跨层契约集中" 原则强于 "stack-by-feature"；放根目录少一次跳转。`base.py` 这个目录就不建了，必要时再加
- **yfinance_provider.py Day 2 不做**：salvaged `stock_data.py` 已经能拿价格；v0.1 三张卡都不需要 OHLCV 时间序列，不必把 yfinance 包成 Provider。Day 2 任务降一项更现实
- **EDGAR ticker→CIK 用进程级 cache + lock**：每次新建 EdgarProvider 不重复拉 ~10MB JSON。后续如果换 worker 进程模型再考虑迁到 Redis
- **测试用 `monkeypatch._akshare()` 而非真 import akshare**：测试运行时不需要装 akshare（300+ MB 依赖图），CI 友好

### 遗留

- AKShare 列名假设是 1.15+ 的中文版；实际跑起来如果上游改版，列名容错回路会安静返回空列表（已加 logger.warning），但不会抛 —— 改 v0.5 时考虑加严
- EDGAR Revenue 概念尝试 ["Revenues", "RevenueFromContractWithCustomerExcludingAssessedTax", "SalesRevenueNet"]，覆盖大多数公司但不是全部；遇到偏门公司（保险 / 银行）可能抽不到
- 没写 yfinance_provider 的好处是 v0.2 加 "股价趋势" 卡时再统一设计，避免现在拍脑袋
