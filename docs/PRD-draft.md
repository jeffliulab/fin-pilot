# FinPilot PRD（草稿 v0.1）

> 状态：**草稿**。基于 [初步探讨思路.txt](初步探讨思路.txt) 整理出的初版需求，未经评审。
> 用途：作为简历附件 / 面试展示用，体现"做减法"的产品判断。

## 1. 一句话定位

FinPilot 是面向中国金融行业从业者（投研 / 投顾 / 合规）的 AI Copilot，
通过 RAG + Function Calling + 多 Agent 共识，把每天 30 份研报、监管新规、
碎片化行情压缩成可执行的工作产出。

**类比**：GitHub Copilot 之于程序员。

## 2. 不做什么（避免 scope creep）

- ❌ 不做财务记账 / 报税 / CFO 工作流（→ `agent-as-a-cfo`）
- ❌ 不做交易执行 / 量化回测（合规 + 牌照风险）
- ❌ 不做"具体产品推荐"（非持牌投顾红线，模块 4 仅给配置框架）

## 3. 模块取舍

| # | 模块 | 优先级 | Demo 是否实现 |
|---|------|--------|--------------|
| 1 | 个股 / 行业深度分析 | P0 | ✅ |

## 4. 模块 1 — 个股 / 行业深度分析

**目标用户**：投研分析师、买方研究员
**核心能力**：
1. 输入公司名称或股票代码 → Agent 自动抓取年报、研报、公告
2. 输出标准化分析摘要（财务、业务、风险）
3. 支持自然语言追问："现金流为什么连续两年下滑？"
4. 支持同行对比："把宁德时代和比亚迪的毛利率趋势画图"

**技术落点**：RAG（长文档分块） + Function Calling（财务指标计算） + 多轮追问规划
**复用资产**：`src/fin_pilot/{llm,analysis,agents,data}/`

## 5. 模块 2 — 销售话术 / 营销物料合规审查

**目标用户**：客户经理、合规专员、产品经理
**核心能力**：
1. 粘贴产品宣传文案 → 基于监管规则库审查
2. 标红违规点（"保本保息"、"年化 10% 起" 等）+ 给出合规修改建议
3. 输出合规评分 + 修改后版本

**技术落点**：Model Court 多 Agent 共识（适当性 / 风险揭示 / 收益表述各一个 Agent） + RAG（监管规则库）
**为什么是灵魂功能**：把已发布的 PyPI 框架（Model Court / ANIMA）直接商用化 ——
"这不是新做的 Demo，这是开源框架在金融场景的落地"。

## 6. 模块 3 — 投研晨报生成（选做，先写 PRD）

8:00 自动汇总昨日公告 / 研报 / 新闻 / 海外行情 → 结构化晨报，按板块分类、按重要性排序，支持订阅。
**技术落点**：多文档处理 + 实体抽取 + 自动分类 + 摘要生成。

## 7. 模块 4 — 客户画像与配置建议（选做，先写 PRD）

KYC + 对话记录 → 客户画像 → "配置框架建议"（**不推具体产品**，避开非持牌投顾红线）。
**技术落点**：结构化信息抽取 + 规则引擎 + Agent 决策。
